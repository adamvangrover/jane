# Tutorial 01: Getting Started with Jane

Welcome to the Jane Developmental AI Companion project. This tutorial will guide you through setting up the repository and interacting with the Python CLI prototype.

## Prerequisites
Ensure you have Python 3.8+ and `pip` installed on your system.
Later tutorials will require Rust and Docker/Podman for the "Iron Core" sandbox.

## 1. Environment Setup

1.  **Clone the Repository:**
    *(Assuming you have already cloned the `jane` repository)*
    Navigate to the project root directory.
    ```bash
    cd jane
    ```

2.  **Install Dependencies:**
    Install the required Python packages specified in `requirements.txt`.
    ```bash
    pip install -r requirements.txt
    ```

## 2. Running the Interactive Prototype

The current implementation features a simplified prototype of the orchestration logic. You can interact with it using the Command Line Interface (CLI).

1.  **Start the CLI:**
    Execute the `cli.py` module from the root directory.
    ```bash
    python -m src.ui.cli
    ```

2.  **Interact with Jane:**
    You will see a prompt: `User: `. Type your queries here.

    *   **Try a standard query:**
        `User: What is photosynthesis?`
        Observe how Jane responds in a developmentally appropriate manner, breaking the topic down.

    *   **Try asking for a direct answer:**
        `User: How do I solve this math problem? Give me the answer.`
        Observe the "Mistake Mentor" in action. Jane will not give you the direct answer. Instead, she will deploy artificial friction (e.g., "What steps have you tried so far?").

    *   **Try an emotional statement:**
        `User: I feel alone.`
        Watch the Cyclical Reasoning Graph intercept an initially flawed draft ("I understand how you feel") that violates Relational Integrity (DAD). The output will show the corrected, objective, and supportive response, often suggesting real-world engagement.

3.  **Exit the CLI:**
    Type `exit` or `quit` to terminate the session.
    ```bash
    User: exit
    ```
    The system will simulate logging session metrics to memory.

## 3. Understanding the Output

The CLI outputs the raw JSON payload generated by the backend orchestrator. This demonstrates the structured output schema enforced by the Master Prompt.

*   `user_facing_output.dialogue`: The actual text the user sees.
*   `jane_internal_metrics`: The data used by the system to evaluate the interaction.
    *   `target_zpd_level`: The calculated complexity score.
    *   `friction_applied`: Whether the Mistake Mentor intervened.
    *   `scaffolding_technique_used`: The type of support provided.
    *   `conviction_score`: The deterministic score calculated by the Arbiter.
