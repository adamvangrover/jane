# Jane

## Project Scaffold

`Jane` is a repository scaffold for a future developmental AI companion and secure agent architecture. It is intended to capture foundational design concepts, behavioral guardrails, and the high-level structure necessary for a human-centered, long-term companion system.

## Repository Purpose

This project is not a production application yet. It exists as a design reference for:

- psychology-informed companion design
- secure, policy-driven agent governance
- developmental scaffolding and learning resilience
- secure backend architecture and prompt orchestration
- long-term memory and adaptive personalization

## What to Read First

- `ARCHITECTURE.md` — the main blueprint for the developmental AI companion.
- `README.md` — project orientation and scaffold guidance.

## Recommended Next Steps

1. Review `ARCHITECTURE.md` to understand the philosophy and system design.
2. Define core policy files such as `SOUL.md`, `AGENTS.md`, `IDENTITY.md`, `TOOLS.md`, and `MEMORY.md`.
3. Build the secure backend as a separate implementation layer, isolating UI from core logic.
4. Design the companion's prompt pipeline to enforce DAD, SDT, ZPD, and Mistake Mentor principles.

## Core Architectural Themes

- Developmentally Aligned Design (DAD)
- Self-Determination Theory (SDT)
- Zone of Proximal Development (ZPD)
- Deliberate friction via Mistake Mentor design
- Secure, zero-trust execution and sandboxing
- Policy-as-Code for deterministic behavior
- Iterative reasoning with validation and structured output

## Project Structure

This repository is designed to support a multidisciplinary implementation strategy.

- `ARCHITECTURE.md`: system blueprint, safety principles, and design intent.
- `README.md`: project summary and scaffold notes.

## Implementation Considerations

The Jane companion should be built with a clear separation of concerns:

- frontend/UI: presentation and user interaction only
- backend/orchestration: prompt logic, state tracking, and policy enforcement
- secure core: cryptographic operations, sandboxing, and execution isolation
- memory subsystem: long-term preference tracking, ZPD metrics, and decision history

## Usage

Use this repository as a design and planning reference. It is best suited for:

- product strategy for developmental AI systems
- secure AI architecture planning
- behavioral and educational technology research
- policy-driven companion design

## Notes

The project is intentionally framed around an ethics-first companion model. The Jane system is imagined as a supportive, non-human sidekick that promotes autonomy, competence, and real-world relatedness.
